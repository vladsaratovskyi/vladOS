# Kernel Entry Walkthrough

Back to the [architecture guide](../architecture.md) or the
[walkthrough index](README.md).

This page covers:

- `src/main.rs`: the production boot entry point
- `src/lib.rs`: the shared no-std kernel library

## `src/main.rs`

### Purpose

`src/main.rs` is the normal kernel binary. The bootloader jumps to the `_start`
symbol generated by `bootloader::entry_point!`, passes `BootInfo`, and the
kernel initializes CPU tables, prints memory diagnostics, triggers a breakpoint
proof, and halts.

### Dependencies

- `core::panic::PanicInfo` for the panic handler argument
- `blog_os::gdt` for GDT/TSS setup
- `blog_os::interrupts` for IDT setup
- `blog_os::memory` for active page-table access
- `blog_os::println` for VGA output
- `blog_os::hlt_loop` for the final halt state
- `bootloader::{entry_point, BootInfo}` for the typed boot entry point
- `x86_64::structures::paging::Translate` for virtual-to-physical diagnostics
- `x86_64::{PhysAddr, VirtAddr}` for address formatting and conversion

### Invariants

- `kernel_main` must never return.
- The panic handler must never return.
- Normal boot must not intentionally double fault or page fault.
- `gdt::init()` must run before handlers that depend on the TSS IST stack.
- `interrupts::init_idt()` must run before `int3()`.
- `memory::init()` must receive `boot_info.physical_memory_offset`, not a
  hard-coded offset.

### Line-By-Line

| Code | Explanation |
| --- | --- |
| `#![no_std]` | Disables Rust's standard library. There is no host operating system, so the kernel uses `core`. |
| `#![no_main]` | Disables Rust's normal `main` startup path. The bootloader enters through `_start`. |
| `use core::panic::PanicInfo;` | Imports the type passed to the custom panic handler. |
| `use blog_os::{gdt, hlt_loop, interrupts, memory, println};` | Imports shared kernel setup, memory, halt, and VGA output helpers. |
| `use bootloader::{entry_point, BootInfo};` | Imports the bootloader macro and the structure passed from the bootloader to the kernel. |
| `use x86_64::{structures::paging::Translate, PhysAddr, VirtAddr};` | Imports paging translation and address types for the memory diagnostics. |
| `entry_point!(kernel_main);` | Generates the externally visible `_start` symbol and type-checks that `kernel_main` has signature `fn(&'static BootInfo) -> !`. |
| `fn kernel_main(boot_info: &'static BootInfo) -> !` | Defines the real production kernel entry point. `BootInfo` contains the memory map and physical-memory offset. |
| `println!("Hello from Rust OS!");` | Writes the first visible boot message to VGA text memory. |
| `gdt::init();` | Initializes the TSS, GDT, code segment selector, and TSS selector. |
| `interrupts::init_idt();` | Builds and loads the production IDT with exception handlers. |
| `let physical_memory_offset = VirtAddr::new(boot_info.physical_memory_offset);` | Converts the bootloader-provided offset into the `x86_64` virtual-address type. |
| `let mapper = unsafe { memory::init(physical_memory_offset) };` | Creates an `OffsetPageTable` for the active page tables. This is unsafe because the offset must match the bootloader mapping. |
| `print_memory_diagnostics(...)` | Prints the current mapping strategy and a few selected virtual-to-physical translations without changing mappings. |
| `x86_64::instructions::interrupts::int3();` | Executes `int3`, deliberately raising the breakpoint exception on vector 3. |
| `println!("Still alive after breakpoint");` | Proves the breakpoint handler returned normally. |
| `hlt_loop();` | Halts forever. Normal boot does not exit on its own. |
| `#[panic_handler]` | Marks the function that handles panics in this `no_std` binary. |
| `fn panic(info: &PanicInfo) -> !` | Receives panic details and never returns. |
| `println!("{}", info);` | Prints panic information through VGA output. |
| `hlt_loop();` | Halts forever after a panic. |
| `fn print_memory_diagnostics(...)` | Keeps the boot function compact and groups the read-only memory diagnostics. |
| `boot_info.memory_map.iter().filter(...Usable).count()` | Counts the memory regions that the bootloader says the kernel may allocate from. |
| `println!("  physical_memory_offset: {:?}", physical_memory_offset);` | Prints the runtime direct-map offset chosen by the bootloader. |
| `print_translation(mapper, "VGA", VirtAddr::new(0xb8000));` | Shows where VGA text memory currently maps. |
| `print_translation(... "kernel_main" ...)` | Shows where the kernel entry function maps. The function item is cast through a pointer before becoming an integer address. |
| `print_translation(mapper, "BootInfo", VirtAddr::from_ptr(boot_info));` | Shows where the boot information structure maps. |
| `fn print_translation(...)` | Uses the mapper's `translate_addr` helper and prints either a physical address or `unmapped`. |
| `fn print_mapping(...)` | Formats one successful virtual-to-physical mapping line. |

## `src/lib.rs`

### Purpose

`src/lib.rs` is the shared kernel crate. The production binary and integration
test kernels import common modules from here.

### Invariants

- This crate must remain `no_std`.
- It must expose only reusable kernel pieces, not production-only `kernel_main`.
- `hlt_loop()` must never return.

### Line-By-Line

| Code | Explanation |
| --- | --- |
| `#![no_std]` | Keeps the library compatible with bare-metal kernel code. |
| `#![feature(abi_x86_interrupt)]` | Enables the nightly interrupt ABI used by exception handlers and tests. |
| `pub mod gdt;` | Exposes GDT/TSS setup. |
| `pub mod interrupts;` | Exposes the production IDT setup and exception handlers. |
| `pub mod memory;` | Exposes active page-table access and the boot-info frame allocator. |
| `pub mod qemu;` | Exposes QEMU debug-exit helpers used by integration tests. |
| `pub mod serial;` | Exposes COM1 serial output used by tests. |
| `pub mod vga_buffer;` | Exposes VGA text output and the `print!`/`println!` macros. |
| `pub fn hlt_loop() -> !` | Defines the shared halt loop. It never returns. |
| `loop { x86_64::instructions::hlt(); }` | Repeatedly sleeps the CPU until the next interrupt instead of busy-spinning. |
