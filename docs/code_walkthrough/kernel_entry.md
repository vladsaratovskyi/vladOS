# Kernel Entry Walkthrough

Back to the [architecture guide](../architecture.md) or the
[walkthrough index](README.md).

This page covers:

- `src/main.rs`: the production boot entry point
- `src/lib.rs`: the shared no-std kernel library

## `src/main.rs`

### Purpose

`src/main.rs` is the normal kernel binary. The bootloader jumps to the `_start`
symbol generated by `bootloader::entry_point!`, passes `BootInfo`, and the
kernel initializes CPU tables, prints memory diagnostics, initializes the fixed
heap, enables the legacy timer and keyboard interrupt path, triggers a
breakpoint proof, runs a short cooperative task demo, and halts. Preemptive
task switching is available for tests and later demos, but normal boot does not
enable it yet. The minimal userspace, address-space isolation, and embedded ELF
loader paths are exercised by `tests/userspace.rs`, `tests/address_spaces.rs`,
and `tests/elf_loader.rs` instead of the normal boot demo.

### Dependencies

- `core::panic::PanicInfo` for the panic handler argument
- `vlad_os::allocator` for fixed heap setup
- `vlad_os::gdt` for GDT/TSS setup
- `vlad_os::interrupts` for IDT, PIC, PIT, and interrupt enable setup
- `vlad_os::memory` for active page-table access
- `vlad_os::println` for VGA output
- `vlad_os::scheduler` for spawning and running the cooperative task demo
- `vlad_os::hlt_loop` for the final halt state
- `bootloader::{entry_point, BootInfo}` for the typed boot entry point
- `x86_64::structures::paging::Translate` for virtual-to-physical diagnostics
- `x86_64::{PhysAddr, VirtAddr}` for address formatting and conversion

### Invariants

- `kernel_main` must never return.
- The panic handler must never return.
- Normal boot must not intentionally double fault or page fault.
- `gdt::init()` must run before handlers that depend on the TSS IST stack.
- `interrupts::init_idt()` must run before PIC initialization and `int3()`.
- The PICs must be remapped and the PIT must be programmed before CPU
  interrupts are enabled.
- `memory::init()` must receive `boot_info.physical_memory_offset`, not a
  hard-coded offset.
- `allocator::init_heap()` must run before any kernel code uses `alloc` types.
- Heap initialization needs both the active mapper and the boot-info frame
  allocator.
- Demo tasks are spawned only after the heap is initialized because each task
  owns a heap-backed stack.

### Line-By-Line

| Code | Explanation |
| --- | --- |
| `#![no_std]` | Disables Rust's standard library. There is no host operating system, so the kernel uses `core`. |
| `#![no_main]` | Disables Rust's normal `main` startup path. The bootloader enters through `_start`. |
| `use core::panic::PanicInfo;` | Imports the type passed to the custom panic handler. |
| `use vlad_os::{allocator, gdt, hlt_loop, interrupts, memory, println, scheduler};` | Imports shared kernel setup, heap setup, memory, halt, VGA output, and cooperative scheduling helpers. |
| `use bootloader::{entry_point, BootInfo};` | Imports the bootloader macro and the structure passed from the bootloader to the kernel. |
| `use x86_64::{structures::paging::Translate, PhysAddr, VirtAddr};` | Imports paging translation and address types for the memory diagnostics. |
| `entry_point!(kernel_main);` | Generates the externally visible `_start` symbol and type-checks that `kernel_main` has signature `fn(&'static BootInfo) -> !`. |
| `fn kernel_main(boot_info: &'static BootInfo) -> !` | Defines the real production kernel entry point. `BootInfo` contains the memory map and physical-memory offset. |
| `println!("Hello from vladOS!");` | Writes the first visible boot message to VGA text memory. |
| `gdt::init();` | Initializes the TSS, GDT, code segment selector, and TSS selector. |
| `interrupts::init_idt();` | Builds and loads the production IDT with exception and hardware IRQ handlers. |
| `interrupts::init_pics();` | Remaps the legacy 8259 PICs to vectors outside the CPU exception range and unmasks only timer and keyboard IRQs. |
| `interrupts::init_pit();` | Programs PIT channel 0 as the initial 100 Hz timer source. |
| `let physical_memory_offset = VirtAddr::new(boot_info.physical_memory_offset);` | Converts the bootloader-provided offset into the `x86_64` virtual-address type. |
| `let mut mapper = unsafe { memory::init(physical_memory_offset) };` | Creates an `OffsetPageTable` for the active page tables. It is mutable because heap setup adds new mappings. This is unsafe because the offset must match the bootloader mapping. |
| `print_memory_diagnostics(...)` | Prints the current mapping strategy and a few selected virtual-to-physical translations without changing mappings. |
| `let mut frame_allocator = unsafe { memory::BootInfoFrameAllocator::init(&boot_info.memory_map) };` | Creates the monotonic physical frame allocator from the bootloader memory map. Heap setup uses it for page-table frames and heap backing frames. |
| `allocator::init_heap(&mut mapper, &mut frame_allocator).expect(...)` | Maps the fixed heap range and initializes the global allocator. Boot halts through the panic handler if setup fails. |
| `println!("Heap initialized");` | Prints a concise success message after the allocator is ready. |
| `interrupts::enable_interrupts();` | Sets the CPU interrupt flag after the IDT, PICs, PIT, and early heap are ready. Timer IRQs can now increment the tick counter, and keyboard IRQs can log raw scancodes. |
| `x86_64::instructions::interrupts::int3();` | Executes `int3`, deliberately raising the breakpoint exception on vector 3. |
| `println!("Still alive after breakpoint");` | Proves the breakpoint handler returned normally. |
| `run_demo_tasks();` | Spawns two short cooperative kernel tasks, runs the scheduler, and returns after both tasks finish. |
| `hlt_loop();` | Halts forever. Normal boot does not exit on its own. |
| `#[panic_handler]` | Marks the function that handles panics in this `no_std` binary. |
| `fn panic(info: &PanicInfo) -> !` | Receives panic details and never returns. |
| `println!("{}", info);` | Prints panic information through VGA output. |
| `hlt_loop();` | Halts forever after a panic. |
| `fn print_memory_diagnostics(...)` | Keeps the boot function compact and groups the read-only memory diagnostics. |
| `boot_info.memory_map.iter().filter(...Usable).count()` | Counts the memory regions that the bootloader says the kernel may allocate from. |
| `println!("  physical_memory_offset: {:?}", physical_memory_offset);` | Prints the runtime direct-map offset chosen by the bootloader. |
| `print_translation(mapper, "VGA", VirtAddr::new(0xb8000));` | Shows where VGA text memory currently maps. |
| `print_translation(... "kernel_main" ...)` | Shows where the kernel entry function maps. The function item is cast through a pointer before becoming an integer address. |
| `print_translation(mapper, "BootInfo", VirtAddr::from_ptr(boot_info));` | Shows where the boot information structure maps. |
| `fn print_translation(...)` | Uses the mapper's `translate_addr` helper and prints either a physical address or `unmapped`. |
| `fn print_mapping(...)` | Formats one successful virtual-to-physical mapping line. |
| `fn run_demo_tasks()` | Groups the normal-boot cooperative task demonstration. |
| `scheduler::spawn(demo_task_a)` and `scheduler::spawn(demo_task_b)` | Allocates task objects and dedicated stacks for two demo tasks. |
| `scheduler::run();` | Switches from the boot stack into task stacks until all ready tasks finish. |
| `demo_task_a()` and `demo_task_b()` | Print two short progress lines each and call `scheduler::yield_now()` so the other task can run. |

## `src/lib.rs`

### Purpose

`src/lib.rs` is the shared kernel crate. The production binary and integration
test kernels import common modules from here. It also enables the `alloc` crate
and provides the allocation error handler shared by all bootable kernels.

### Invariants

- This crate must remain `no_std`.
- It must expose only reusable kernel pieces, not production-only `kernel_main`.
- `hlt_loop()` must never return.

### Line-By-Line

| Code | Explanation |
| --- | --- |
| `#![no_std]` | Keeps the library compatible with bare-metal kernel code. |
| `#![feature(abi_x86_interrupt)]` | Enables the nightly interrupt ABI used by exception handlers and tests. |
| `#![feature(alloc_error_handler)]` | Enables the no-std allocation failure hook required by the `alloc` crate. |
| `extern crate alloc;` | Makes Rust's heap-allocated collection crate available without enabling `std`. |
| `pub mod address_space;` | Exposes isolated user page-table roots and eager user-page mapping helpers. |
| `pub mod allocator;` | Exposes fixed heap mapping and global allocator setup. |
| `pub mod arch;` | Exposes architecture-specific code, currently the x86_64 context switch. |
| `pub mod elf;` | Exposes the strict embedded ELF64 loader used by process-style user task spawning. |
| `pub mod gdt;` | Exposes GDT/TSS setup. |
| `pub mod interrupts;` | Exposes the production IDT setup, exception handlers, PIC/PIT initialization, IRQ indexes, and timer tick counter. |
| `pub mod memory;` | Exposes active page-table access and the boot-info frame allocator. |
| `pub mod qemu;` | Exposes QEMU debug-exit helpers used by integration tests. |
| `pub mod scheduler;` | Exposes task spawning, running, explicit yielding, preemption control, and test/debug status helpers. |
| `pub mod serial;` | Exposes COM1 serial output used by tests. |
| `pub mod syscall;` | Exposes the minimal `int 0x80` syscall numbers and dispatcher used by the syscall interrupt path. |
| `pub mod task;` | Exposes the task ID, task states, task stack size, and task cap. |
| `pub mod user;` | Exposes the tiny user programs and mapping helpers used by the userspace integration test. |
| `pub mod vga_buffer;` | Exposes VGA text output and the `print!`/`println!` macros. |
| `pub fn hlt_loop() -> !` | Defines the shared halt loop. It never returns. |
| `loop { x86_64::instructions::hlt(); }` | Repeatedly sleeps the CPU until the next interrupt instead of busy-spinning. |
| `#[alloc_error_handler]` | Marks the function called when the global allocator cannot satisfy an allocation. |
| `panic!("allocation error: {:?}", layout)` | Reports the failed allocation layout through the active panic handler. |
